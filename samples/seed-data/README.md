# seed-data

HSQL in-memory schema + seed data. **Every runner** includes this directory
regardless of jdk/framework choice (it's in `alwaysInclude` on the matrix).

## Files

| File | Purpose |
|---|---|
| `schema.sql` | 6 tables + indexes + FKs |
| `data.sql`   | Seed rows (users / dept / board / wide / file / 1000× large) |

## Tables (6)

| # | Table          | Rows after seed | Purpose                                   |
|---|----------------|-----------------|-------------------------------------------|
| 1 | `USERS`        | 3               | Stub login + session demo                 |
| 2 | `SAMPLE_BOARD` | 5               | CRUD + `_RowType_` N/I/U/D/O exercise     |
| 3 | `DEPT`         | 7               | Parent-child tree (TreeView demo)         |
| 4 | `LARGE_DATA`   | 1000            | Paging (`/large/page.do`) + streaming (`/large/stream.do`) |
| 5 | `WIDE_COLUMNS` | 1               | Wide-column load (`/wide/load.do`) — 50 cols |
| 6 | `FILE_META`    | 2 metadata rows | Upload / download metadata (`/file/*`)    |

## How each runner loads this

### Spring Boot runners (boot / egov-boot / webflux)

Spring Boot auto-loads `schema.sql` + `data.sql` from the classpath when
`spring.sql.init.mode=always` is set. Runners in Plan 2 will either:

- Copy these two files to `src/main/resources/` at scaffold time, OR
- Reference them via `spring.sql.init.schema-locations=classpath:seed-data/schema.sql`

### Spring MVC runners (mvc / egov-mvc)

A `@PostConstruct` listener runs both scripts via `ScriptUtils.executeSqlScript`
at war startup.

## Dialect notes (HSQL 2.x)

- `GENERATED BY DEFAULT AS IDENTITY` — HSQL's syntax for auto-increment
- `CLOB` type is native
- `BOOLEAN` is native (unlike Oracle/DB2 which need NUMBER(1))
- Recursive CTE (`WITH RECURSIVE`) is supported since HSQL 2.3

If the target engine changes (PostgreSQL / Oracle / MySQL), these scripts
need dialect translation — tracked in a future Plan.
